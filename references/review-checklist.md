# Review Checklist

用于用户要求检查已有协作文档时。评估标准是：另一个开发者或另一个 Codex 能否直接开工。

## 输出顺序

先列阻塞开发的问题，再列建议优化项。不要只评价“文档完整”，要明确指出会让开发卡住的问题。

## 重点检查

- 文档阶段是否明确：开发前、开发中、开发后恢复/复盘、提测/发布。
- `PLAN.md` 的范围是否和其他文件一致。
- skill 或模板示例中是否夹带了某个具体项目的字段名、业务模式、枚举值、表名、接口路径或响应对象。
- `BUSINESS-RULES.md` 是否足够详细、可执行、可验证。
- `API-CONTRACT.md` 是否包含字段类型、必填、ID/名称、错误场景和示例。
- `API-CONTRACT.md` 中对象名是否能跳转到对象定义，且每个接口是否有独立字段视图，而不是只引用通用 VO。
- 涉及数据库变更时，是否有项目迁移脚本、发布/回滚说明，或已确认新增的 `DATA-MODEL.md` 承接 DDL、索引、回填和回滚。
- 文档包是否静默新增了用户未选择、未确认的文件。
- `TASK-BOARD.md` 是否能支撑多人分工，并能看出进度、负责人、依赖和阻塞。
- 团队人数、负责人、排期、优先级是否来自用户明确确认；未确认时是否写成了待确认或待分配。
- 即使负责人待确认，任务包边界是否足够清楚，开发者是否能直接认领任务包开工。
- `TEST-CASES.md` 是否覆盖关键规则、异常和边界，并能映射回业务规则和接口。
- 关键业务规则、接口、任务、测试之间是否有追踪编号或覆盖矩阵。
- 关联编号是否都是可点击链接，且目标锚点存在。

## 范围冲突

检查这些典型冲突：

- `PLAN.md` 说本期不做，但接口、测试或任务拆分里写成本期要做。
- `PLAN.md` 说附件、编辑、删除、发布等放后续，但其他文档写了当前实现规则。
- 当前阶段和后续阶段没有标记，导致开发者无法判断是否要做。

## 接口契约问题

检查这些典型缺口：

- 只有字段名，没有字段类型、必填、含义和示例。
- 只写 `请求字段：某某 VO` 或 `响应业务数据：某某 VO`，没有本接口字段表。
- 只写 `字段见对象定义`，导致读者必须跳到通用对象区才能知道当前接口返回什么字段。
- 通用对象中写了统一必填，但没有说明不同接口下的必填差异。
- 对象名不可点击，开发者需要自己去找对象定义。
- 对象名虽然可点击，但接口正文附近没有就近字段明细，阅读和联调仍然不方便。
- 对象链接被写进反引号代码片段，渲染后看起来像文本但无法点击。
- 把某个项目的统一响应结构、业务数据字段或分页字段名写成全局默认模板。
- 不清楚字段传 ID 还是名称。
- 不清楚字段保存到哪个表字段。
- 不清楚错误响应和业务校验失败提示。
- 不清楚多字段冲突时谁优先。
- 只写“兼容旧字段”，没有说明新前端是否可以传、两个字段同时存在时谁优先、哪些业务模式不能兜底。
- 不清楚附件、分类、标签、回填等复杂结构。
- 没有接口编号，无法分派给具体后端任务。
- 没有对应业务规则、任务或测试用例，无法判断是否漏实现。

## 业务规则问题

检查这些典型缺口：

- 规则只写“需要校验”，没有写校验条件和失败结果。
- 复杂表单只有矩阵摘要，没有写类型切换、字段显隐、动态标签、动态附件等特殊操作流。
- 没有说明默认值、排序、筛选、去重、空数据。
- 没有说明新增、编辑、删除、回填是否同一套规则。
- 没有说明字典名称变化后按名称还是固定 ID 判断。
- 没有按页面、字段、业务模式、操作类型拆分规则。
- 只有规则摘要，没有失败提示、触发时机、前端拦截和后端兜底说明。
- 把 `ALTER TABLE`、`CREATE INDEX`、迁移 SQL、回填 SQL 或回滚 SQL 写进了 `BUSINESS-RULES.md`。

## 数据库变更问题

检查这些典型缺口：

- 有新增字段或索引，但没有说明字段类型、长度、默认值、是否允许空和注释。
- 有业务筛选字段，但没有说明旧数据兼容或回填策略。
- 有查询口径变化，但没有说明是否需要索引或性能验证。
- 有 DDL，但没有上线顺序、回滚方案或验证 SQL。
- 数据库变更没有映射到后端任务、测试用例和发布检查。
- 未经用户确认突然新增 `DATA-MODEL.md`，导致文档包清单和前期约定不一致。

## 任务和测试问题

检查这些典型缺口：

- 任务拆分太粗，不能避免多人重复开发。
- 没有负责人、范围、依赖、完成标准。
- 没有进度状态，看不出未开始、开发中、联调中、阻塞或已完成。
- 两个前端、两个后端等多人场景仍只写“前端负责”“后端负责”。
- 测试用例没有前置条件、步骤、期望结果。
- 测试没有覆盖权限、边界、异常、字典变化、范围排除项。
- 测试用例没有编号，无法追踪到业务规则或接口。
- 关联编号只是纯文本，或用范围写法导致无法逐项跳转。
